# AI Image Platform - Объяснение для новичков

Добро пожаловать! Этот документ объясняет структуру проекта простым языком.

## Что такое этот проект?

Представь, что ты строишь дом из конструктора Lego. Наш проект — это точно так же, только вместо пластиковых кубиков мы используем код.

Этот проект — **веб-сайт**, на котором люди смогут:
- Генерировать картинки с помощью искусственного интеллекта (AI)
- Редактировать эти картинки прямо в браузере
- Использовать готовые шаблоны для создания стикеров, визиток и баннеров

## Технологии (простым языком)

### Next.js 15
Это **фреймворк** (готовый набор инструментов) для создания сайтов. Как готовая кухня, где уже есть плита, холодильник и посуда — тебе остается только готовить.

**Почему Next.js?**
- Быстрый и современный
- Автоматически делает сайт быстрее
- Легко выкладывать в интернет

### React 19
Это **библиотека** для создания интерфейсов. Представь React как набор готовых деталей (кнопки, формы, карточки), которые ты можешь собирать в нужном порядке.

### TypeScript
Это улучшенная версия JavaScript (языка программирования для веба). TypeScript помогает **избегать ошибок**, потому что заранее проверяет, правильно ли ты написал код.

**Пример:**
```typescript
// TypeScript сразу скажет, если ты попытаешься сложить число и текст
let age: number = 25;
age = "двадцать пять"; // ❌ Ошибка! age должен быть числом
```

### Tailwind CSS
Это способ делать сайт красивым. Вместо того чтобы писать длинные CSS-файлы, ты просто добавляешь классы прямо в HTML:

```tsx
<button className="bg-blue-500 text-white p-4 rounded">
  Нажми меня
</button>
```

Это создаст синюю кнопку с белым текстом, отступами и скругленными углами.

### shadcn/ui
Набор **готовых красивых компонентов** (кнопки, формы, модальные окна), чтобы не рисовать все с нуля. Как готовые детали Lego, которые ты просто вставляешь в проект.

### Что такое "компонент"?

**Компонент** — это переиспользуемая часть интерфейса. Например, кнопка "Нравится" на Facebook — это компонент. Она используется везде: под постами, в комментариях, в Stories.

Вместо того чтобы копировать код кнопки 100 раз, ты создаешь один компонент `LikeButton` и используешь его везде.

## Основные файлы

### `package.json`
Список всех библиотек (зависимостей), которые нужны проекту. Как список покупок для супермаркета.

### `tsconfig.json`
Настройки TypeScript. Говорит компилятору, как проверять твой код.

### `tailwind.config.ts`
Настройки Tailwind CSS: цвета, шрифты, отступы.

### `next.config.ts`
Настройки Next.js: какие картинки можно загружать, как обрабатывать файлы и т.д.

## Основные концепции Next.js

### App Router (новая система маршрутизации)

В папке `app/` каждая подпапка создает новый URL:

```
app/
├── page.tsx           → /
├── about/
│   └── page.tsx       → /about
└── blog/
    └── page.tsx       → /blog
```

### Server Components vs Client Components

**Server Components** (по умолчанию):
- Выполняются на сервере
- Быстрее загружаются
- Не могут использовать интерактивность (onClick, useState)

**Client Components** (нужно добавить `'use client'`):
- Выполняются в браузере
- Могут использовать хуки React (useState, useEffect)
- Нужны для кнопок, форм, анимаций

```tsx
'use client'; // Это делает компонент клиентским

import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <button onClick={() => setCount(count + 1)}>
      Счетчик: {count}
    </button>
  );
}
```

## Часто задаваемые вопросы

**Q: Что такое node_modules и почему там так много файлов?**
A: Это папка со всеми библиотеками и их зависимостями. Представь, что ты скачал одну программу, но ей нужны еще 10 других программ для работы — все они попадут в `node_modules`.

**Q: Нужно ли коммитить node_modules в Git?**
A: Нет! Эта папка огромная и есть в `.gitignore`. Другие разработчики скачают зависимости через `npm install`.

**Q: Что делать, если что-то сломалось?**
A: 
1. Попробуй перезапустить сервер (Ctrl+C, затем `npm run dev`)
2. Удали папку `node_modules` и файл `package-lock.json`, затем `npm install`
3. Почитай ошибку в терминале — часто там написано, что не так

## Глоссарий (словарь терминов)

- **Компонент** — переиспользуемая часть интерфейса (кнопка, карточка, форма)
- **Props** — данные, передаваемые в компонент (как аргументы функции)
- **State** — состояние компонента (данные, которые могут меняться)
- **Hook** — специальная функция React (useState, useEffect)
- **API Route** — серверная функция для обработки запросов
- **Middleware** — код, который выполняется между запросом и ответом
- **SSR** — Server-Side Rendering (рендеринг на сервере)
- **CSR** — Client-Side Rendering (рендеринг в браузере)
